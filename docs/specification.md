# CsStructureViewer 仕様書

## 1. 目的・背景

バイブコーディング（AI支援によるコーディング）の過程で、AIによる実装の設計を可視化・チェックするためのツール。  
C# および Unity プロジェクトにおけるクラス構造と依存関係を視覚的に把握することを目的とする。

---

## 2. 対象プロジェクト

- C# プロジェクト（.NET）
- Unity プロジェクト（C# スクリプト）

---

## 3. フレームワーク・技術スタック

| 項目 | 採用技術 |
|------|---------|
| UIフレームワーク | WPF + .NET 10 |
| コード解析 | Roslyn（Microsoft.CodeAnalysis.CSharp） |
| 配布形式 | 単一実行ファイル（self-contained、セットアップ不要） |

---

## 4. ビルド・開発運用仕様

### 4.1 ビルド成果物

- 修正作業のたびにビルドを実行し、`publish/` フォルダに `.exe` ファイルを生成・更新する。
- ビルド成果物はユーザーがすぐ実行確認できる状態に保つ。

### 4.2 開発中のテスト対象

- 開発中の動作確認では、リポジトリ直下の `DebugProject/` フォルダを解析対象プロジェクトとして使用する。
- 解析・表示・更新など、対象プロジェクトを必要とする機能の確認は原則として `DebugProject/` に対して行う。

## 5. 表示仕様

### 5.1 クラス矩形

- クラスを矩形で表現し、矩形内にクラス名を表示する
- 矩形のサイズはクラス名の長さに応じて可変とする
- 最大幅を設定し、それを超えるクラス名は折り返して表示する
- 塗りつぶし色は所属する名前空間矩形と同系色とする
- ネストクラスは、外側クラスの矩形内に子クラス矩形を入れ子で表示する

### 5.2 依存関係の矢印

Roslynで解析した依存関係をUML矢印スタイルで表示する。

| 関係の種類 | 検出方法 | UML表現 |
|-----------|---------|--------|
| 継承 | `: BaseClass` 宣言 | 汎化矢印（実線＋塗りつぶし三角） |
| インターフェース実装 | `: IInterface` 宣言 | 実現矢印（破線＋空三角） |
| フィールド・プロパティ型参照 | フィールド・プロパティの型宣言 | 関連矢印（実線＋開き矢印） |

**矢印のルーティング：**

- 矢印は直角折れ線（オルソゴナルルーティング）で描画する
- 矢印の経路はすべての矩形（クラス矩形・名前空間矩形・フォルダ矩形）を避ける
- 矢印の接続辺は、中心点の位置関係だけで固定せず、複数の接続辺候補を評価して選択する
  - 候補ごとに経路を生成し、線分長・曲がり回数・矩形との交差有無を基に、より短く曲がりの少ない自然な経路を採用する
  - 評価時は、矩形を避けることを最優先し、その次に曲がり回数の少なさ、線分長の短さを優先する
  - 接続辺は実際の経路方向と整合させる。下方向へ出た経路で同じ高さの矩形へ向かう場合は、相手矩形にも下側から入るなど、出射方向と入射方向が不自然に反転しない組み合わせを選ぶ
- 同じ辺を通過する複数の矢印は、辺上で均等にずらして重ならないように表示する
  - 辺への入射・出射いずれの矢印も対象とする
  - 同一の始点矩形または終点矩形から複数の矢印が出る場合も同様にずらす
  - ずらし量は辺ごとに独立して計算する。例えば矩形の4点が `(0,0)`, `(1,0)`, `(0,1)`, `(1,1)` の場合、下辺に1本入射する矢印の終点は `(0.5, 1)` とする。下辺に2本入射する場合は `(0.333..., 1)`, `(0.666..., 1)` とする。上辺に1本・下辺に1本入射する場合は、それぞれ `(0.5, 0)`, `(0.5, 1)` とする
- 矢印の入射・出射方向は接続辺と一致させる
  - 上辺へ入る矢印は上から下へ進入する
  - 下辺へ入る矢印は下から上へ進入する
  - 左辺へ入る矢印は左から右へ進入する
  - 右辺へ入る矢印は右から左へ進入する
  - 上辺から出る矢印は下から上へ出る
  - 下辺から出る矢印は上から下へ出る
  - 左辺から出る矢印は右から左へ出る
  - 右辺から出る矢印は左から右へ出る
- 矢印線同士の交差は許容するが、線分が完全に重なり合う状態は許容しない

### 5.3 名前空間矩形

- `namespace` 宣言を基準に名前空間を決定する（フォルダ構造は使用しない）
- 矩形の枠上に名前空間名を表示する（例: `-- MyNamespace ------`）
- 矩形は半透明の塗りつぶしで表示する
- 名前空間ごとに色を自動割り当てる（隣接する名前空間と一定以上の色差を確保）
- 名前空間矩形の中に、その名前空間に属するクラス矩形を配置する
- 通常の名前空間・クラスは、矢印が入る・出るのどちらもない孤立状態であっても表示する

### 5.4 グローバル名前空間のクラス

`namespace` 宣言がないクラスは名前空間矩形を持たず、背景に直接クラス矩形を表示する。

### 5.5 パーシャルクラス

`partial class` として複数ファイルに分散して定義されているクラスは、1つのクラス矩形にまとめて表示する。

### 5.6 フォルダ矩形（内部依存除外）

「内部依存除外パターン」に一致する名前空間は、通常の名前空間矩形とは異なるフォルダ矩形として表示する。

**表示仕様：**

- 枠線は点線、塗りつぶしはグレーの半透明
- 枠上のラベルは通常の名前空間矩形と同じ形式（例: `-- Library ------`）
- フォルダ矩形の中には名前空間矩形のみを含める。クラス矩形は表示しない
- フォルダ矩形のサイズは、非表示にするクラス矩形ではなくラベル表示に必要な最小サイズで決定する
- フォルダ矩形内の名前空間同士の矢印も表示しない
- 内部依存除外パターン内部で完結する矢印は表示しない
- 内部依存除外パターンに含まれるクラスが外部（内部依存除外パターン以外の通常クラス）との依存関係を持つ場合、内部依存除外対象の名前空間矩形と通常のクラス矩形の間に矢印を表示する
- 内部依存除外対象と外部の間の矢印は、内部依存除外対象の名前空間・外部クラス・矢印の種類・向きが同じものを1本に集約する
- 外部と矢印のつながりがない内部依存除外対象の名前空間矩形は描画しない

## 6. レイアウト仕様

### 6.1 基本方針

- 名前空間矩形は、内包するクラス矩形を包むサイズに自動計算する
- クラス矩形は名前空間内で横方向に並べ、最大幅を超えたら折り返す
- 名前空間矩形・フォルダ矩形は左上から横方向に並べ、最大幅を超えたら折り返す
- 名前空間間・クラス間それぞれに最小余白（minPadding）を設定する
- クラス間の最小余白は、矢印線が隣接クラス矩形に隠れないよう十分に確保する。現在の既定値は30pxとする
- キャンバスの最大幅は、解析開始時のウィンドウ幅に合わせる

## 7. 操作仕様

### 7.1 プロジェクトを開く

- メニューバーの「開く」を選択するとフォルダ選択ダイアログが表示される
- 選択したフォルダ配下の `.cs` ファイルを解析対象とする
- 完全除外設定に該当するファイル・フォルダは解析から除外する

### 7.2 ビュー操作

| 操作 | 動作 |
|------|------|
| マウスホイール | ズームイン・アウト |
| 左クリック ＋ ドラッグ | パン（ビューの移動） |

- クラス矩形・名前空間矩形のクリックによるアクションは現状なし

### 7.3 解析中の表示

- スピナー（ぐるぐる回るインジケーター）を表示する
- 解析を中止するキャンセルボタンを合わせて表示する

### 7.4 更新

- ビューワーエリアの左上（メニューバーの下）に「更新」ボタンを配置する
- 「更新」ボタンを押すと、現在開いているフォルダを対象に再解析・再描画を行う
- プロジェクト未選択時（起動直後）は「更新」ボタンを非表示にする

## 8. 設定仕様

### 8.1 完全除外パターン

- 設定はアプリケーションのグローバル設定として1つのみ管理する（プロジェクトごとではない）
- 設定画面でフォルダ名・ファイル名のパターンを管理する
- このパターンに一致するファイル・フォルダはRoslyn解析から完全に除外する

**デフォルト：**

```
bin, obj, .git, Editor, Temp, temp, Tests
```

### 8.2 内部依存除外パターン

- 設定画面で名前空間名のパターンを管理する
- このパターンに一致する名前空間はフォルダ矩形として表示される（4.6節参照）
- クラス矩形は非表示とする。外部との依存関係は、内部依存除外対象の名前空間矩形と通常クラス矩形の間の矢印として表示する
- デフォルトは空（パターンなし）

### 8.3 設定画面のUI操作

- 完全除外パターン・内部依存除外パターンはそれぞれリスト形式で表示する
- 各行に `+`（追加）ボタンと `-`（削除）ボタンを表示する
- `+` を押すと、その行の下に新しい入力欄が追加され、編集可能になる
- `-` を押すと、その行を削除する
- パターン未入力の行は保存時に無視する

## 9. UI仕様

### 9.1 メニューバー

| メニュー | 項目 | 動作 |
|---------|------|------|
| ファイル | 開く | フォルダ選択ダイアログを表示し、プロジェクトを読み込む |
| 設定 | 設定画面を開く | 除外設定などを管理する設定画面を開く |

### 9.2 起動時の表示

プロジェクト未選択時は、中央に案内メッセージを表示する。

```
メニューの「ファイル → 開く」からフォルダを選択してください
```

### 9.3 「更新」ボタン

- ビューワーエリアの左上（メニューバーの真下）に配置する
- プロジェクト選択後（解析完了後）のみ表示する
- ラベル：「更新」
