# CsStructureViewer

CsStructureViewer は、C# / Unity プロジェクト内のクラス構造と依存関係を解析し、グラフとして可視化するWindows向けデスクトップアプリケーションです。

コードを一つずつ追わなくても、クラスや名前空間のまとまり、継承、インターフェース実装、フィールド・プロパティ参照の関係を俯瞰できます。解析はローカル環境で完結します。

## 主な機能

- C#ファイルをRoslynで解析
- クラス単位／名前空間単位の表示切り替え
- 継承、インターフェース実装、フィールド・プロパティ参照を矢印で表示
- 名前空間ごとのグループ表示と自動配色
- ネストクラス、パーシャルクラスに対応
- マウスによるズームと表示位置の移動
- フォルダ名・名前空間名による除外設定
- 解析中のキャンセルと、同じフォルダの再解析

## 動作環境

- Windows
- [.NET 10 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/10.0)

本アプリはフレームワーク依存形式で配布されるため、実行するPCに.NET 10 Desktop Runtimeが必要です。

## 導入方法

1. GitHubの[Releases](../../releases)を開きます。
2. 最新リリースの配布用ZIPファイルをダウンロードします。
3. ZIPファイルを任意のフォルダへ展開します。
4. 展開したフォルダ内の `CsStructureViewer.exe` を実行します。

インストーラーはありません。不要になった場合は、展開したフォルダを削除してください。

> [!IMPORTANT]
> リポジトリの「Source code」ZIPには実行ファイルが含まれません。Releasesに添付された配布用ZIPを使用してください。また、`CsStructureViewer.exe` だけを取り出さず、展開されたDLLやJSONファイルと同じフォルダに置いたまま実行してください。

Windowsによって実行が保護された場合は、ダウンロード元がこのリポジトリであることを確認してから、Windowsの案内に従って実行してください。

## 基本的な使い方

1. `CsStructureViewer.exe` を起動します。
2. 画面上部の「ファイル」を選択します。
3. 解析したいC#プロジェクトまたはUnityプロジェクトのルートフォルダを選択します。
4. フォルダ配下の `.cs` ファイルが解析され、依存関係グラフが表示されます。

Unityプロジェクトの場合は、通常は `Assets` フォルダそのものではなく、`Assets` を含むUnityプロジェクトのルートフォルダを選択します。

### グラフの操作

| 操作 | 動作 |
|------|------|
| マウスホイール | ズームイン／ズームアウト |
| 左クリックしながらドラッグ | 表示位置を移動 |
| 「クラス／名前空間」スイッチ | 表示単位を切り替えて再解析 |
| 「更新」ボタン | 現在選択しているフォルダを再解析 |
| 解析中の「キャンセル」ボタン | 解析を中止 |

### 矢印の見方

| 表示 | 意味 |
|------|------|
| 通常の矢印 | フィールド・プロパティ参照 |
| 白い三角形の実線矢印 | 継承 |
| 白い三角形の破線矢印 | インターフェース実装 |

「クラス」表示ではクラスごとの関係を表示します。「名前空間」表示ではクラスを省略し、異なる名前空間間の依存関係を集約して表示します。

## 除外設定

画面上部の「設定」から、解析・表示対象を調整できます。

### 完全除外パターン

一致したフォルダまたは名前空間を解析対象から除外します。初期状態では、次のパターンが登録されています。

```text
bin, obj, .git, Editor, Temp, temp, Tests
```

### 内部依存除外パターン

一致した範囲のクラスを個別表示せず、フォルダ／名前空間単位にまとめて表示します。大きなライブラリや、詳細を追う必要がない領域を折りたたむ用途に利用できます。初期値は `Library` です。

各パターンでは、「フォルダ」「名前空間」のどちらに適用するかを選択できます。通常の文字列は完全一致、末尾に `.*` を付けた文字列は前方一致として扱われます。

例：`UnityEngine.*` は `UnityEngine` と、その配下の名前空間・フォルダ名に一致します。

## 設定と診断情報の保存場所

設定はユーザーごとに次のファイルへ保存されます。

```text
%APPDATA%\CsStructureViewer\settings.json
```

解析時には、実行ファイルと同じ場所に次の診断ファイルが生成されます。

```text
layout-diagnostics\latest-layout.json
```

この診断ファイルには、解析対象フォルダのパス、名前空間名、クラス名、レイアウト情報が含まれます。ログを共有するときは内容を確認してください。

## ソースコードから実行する

.NET 10 SDKをインストールした環境で、リポジトリのルートから次を実行します。

```powershell
dotnet restore
dotnet run
```

配布用ファイルを生成する場合は、次を実行します。

```powershell
dotnet publish -c Release
```

成果物は `publish/` に出力されます。実行時は、出力されたファイル一式を同じフォルダに配置してください。
